SASS - Avantages

Nesting : Imbrication des blocs sans avoir des sélecteurs à rallonge.

Variables : Génériser certaines valeurs, comme les couleurs d'un thème.

Mixins : Regroupement des préfixes vendeurs, simplification d'écritures longues et répétitives.

Fonctions : Tout un panel de fonctions utilisables pour simplifier des traitements.

SASS - Syntaxe


@mixin box-shadow($what : 1px 1px 2px black) {
  -webkit-box-shadow: $what;
     -moz-box-shadow: $what;
          box-shadow: $what;
}
.parent-class {
	.child-class {
		background-color : $primary-color;
		@include box-shadow(0px 2px 2px $background-color);
	}
}

CSS - Position

Logique de positionnement d'un bloc.
Valeurs : static (par défaut), relative, absolute, fixed.

Créé un contexte pour les blocs fils et non static.
Le contexte sert à utiliser par exemple, des pourcentages.
Si on utilise width : 50%; la largeur de l'élément sera égale à 50% du parent non static le plus proche.

CSS - Position (suite)

Cas particulier pour les pourcentages :
margin-bottom : 50% ==> Le pourcentage ne sera pas relative à la hauteur, mais à la largeur, ouais ! C'est mal fait !

Le contexte permet aussi d'utiliser z-index.

CSS - Positionnement


Les propriétés de positionnement top/right/bottom/left ont des effets différents selon les valeurs de la propriété position.

static / relative :
Un élément relative se positionne donc relativement à son parent, chaque élément relatif s'empile dans l'ordre donné. Les valeurs de positionnement bougent l'élément en rapport avec la position initiale qu'il devrait avoir.

CSS - Positionnement


absolute :
Un élément absolu se positionne en rapport à son dernier parent non static. Si on dit bottom 0px, l'élément se place en bas de ce parent.

fixed :
Position relative à la fenêtre.

Exemple - Positionnement

Relative
top: 20px; right: 20px;
width: 70%; height: 33%;
Absolute
bottom: 0px; right: 0px;
width: 70%; height: 33%;
Fixed
bottom: 0px; left: 0px;
width: 20%; height: 33%;

CSS - Display

Le display gère la logique d'affichage de l'élément courant. Par exemple sur un élément relative.

Un DIV de base a un display BLOCK, donc chaque DIV passe par dessous les autres.

Un SPAN de base a un display INLINE, donc chaque SPAN suit l'autre sur la même ligne.

CSS - Display (suite)

A savoir qu’un display inline n’est pas redimensionnable alors qu’un block oui.

Pour couper la poire en deux, un élément redimensionnable et qui s’aligne à ses frères aura “display : inline-block”

Différents display existent pour simuler les listes et les tableaux. Un dernier display tout récent nommé “flex” permet de gérer des éléments responsif plus facilement.

Exemple - Display

Block
width: 50%;
height: 20%;
Inline-Block
width: 50%;
height: 20%;
Inline-Block
width: 40%;
height: 20%;
Inline
width: 40%; height: 20%;

CSS - Display Flex

A la même manière que le weight en Android, display flex permet de gérer la largeur d’éléments par pondération. Chaque élément dans un élément en flex sera aligné.

Si un élément parent contient display flex. Alors ses enfants qui utiliseront la propriété flex ( flex : un nombre) pourra définir la pondération en largeur de l’élément. Si flex n’est pas défini, l’élément prendra la place restante pour tenir à la ligne.

CSS - Display Flex

Une astuce avec display flex est l’alignement vertical. Si vous déclarez un bloc parent en “display : flex”. Vous pouvez centrer son bloc enfant en affectant à cet enfant la propriété “margin : auto”.

Exemple - Flex

margin: auto;
width: 50%;
height: 20%;

Exemple - Flex

flex : 2;
width: 50%;
flex : 3;

CSS - Sélecteurs

Permet de savoir sur quoi on place une propriété CSS.
  • Sélecteurs simples : .nom-de-class, #nom-id, a, img, body …
  • Sélecteurs à propriétés : input[type=’text’]
  • Sélecteurs multiples : .une-classe, .une-autre
  • Sélecteurs hièrarchiques :
    .une-classe .un-descendant
    .une-classe > .un-descendant-direct
  • Sélecteurs d’événements : .classe:hover ou .classe:active ou .classe:link ou .classe:visited
  • Sélecteurs de fils : .classe:first-child ou .class:last-child ou .class:nth-child(n) ou .class:nth-child(even) ou .classe:nth-child(odd) ...

Exemple - Sélecteurs avec SASS


.mon-bouton {
  //Quelques propriétés
  display: flex;
  @include transition(all 200ms ease-out);
  &:hover {
    @include box-shadow(0px 3px 1px $shadow-color);
  }
  > span {
    margin: auto;
  }
}

CSS - Trucs sympa

Permet de savoir sur quoi on place une propriété CSS.
  • box-shadow : Créé une ombre en dessous de l’élément - inset, x, y, blur, couleur.
  • transition : Permet de faire des animations - éléments, durée, fonction, délai
  • pointer-events : Permet d’activer/désactiver la possibilité de cliquer à travers l’élément, pratique pour les éléments translucides.
  • user-select : Permet de refuser la sélection sur l’élément pour prévenir des effets du spam clic.

CSS - Trucs sympa

Permet de savoir sur quoi on place une propriété CSS.
  • white-space : En lui passant la valeur nowrap on empèche le saut de ligne.
  • border-radius : Permet des arrondis sur des blocs. 50% pour faire un rond.
  • transform : Permet de faire des rotations/translation/scaling sur les éléments.
  • animation : assisté de keyframe permet de gérer des animations avec précision.
  • opacity : permet de rendre l’élément plus ou moins translucide ainsi que son contenu. (différent d’un background color en rgba)
  • z-index : gère la priorité en profondeur d’un élément.

Optimiser le CSS

2 techniques : Minimifier, Nesting intelligent

Minimifier un CSS consiste à réduire l’écriture d’un élément.

Exemple classique ⇒ margin : 5px 3px 5px 3px (haut droite bas gauche)

On peut l’écrire aussi ⇒ margin : 5px 3px (haut-bas droite-gauche)

On a aussi ⇒ margin : 5px 5px 5px 5px, ça devient ⇒ margin : 5px

Ceci fonctionne avec les border, border-radius et padding.

Nesting intelligent

On doit limiter le Nesting à 3 voir 4 niveaux pour pas avoir des appels longs trop souvent.

On doit aussi d’un autre côté limiter le nombre de selecteurs pour ne pas avoir un CSS à rallonge qui serait long à interpréter.

Conseils divers

Permet de savoir sur quoi on place une propriété CSS.
  • Utiliser la console de dev pour tester des changements rapidement.
  • Pour tester une propriété, “nomDePropriété generator” sur google, ça marche quasiment à tous les coups !
  • N’hésitez pas à consulter la doc de display flex.
  • La doc de ng-animate est pas mal aussi !
  • Pour faire un “switch” le ternaire est utilisable dans
    ng-class=”myBool ? ‘une-classe’ : ‘une-autre’”.
  • Attention au box-sizing de bootstrap.
  • Attention à la priorité de lecture des fichiers CSS et aux override.
  • Avec gulp, les erreurs SASS s’affichent en console et aucun changement est notable sur la page web, consultez la console gulp si vous remarquez qu’un de vos changements n’a pas été effectué. Une erreur de syntaxe a simplement bloqué la compilation.
  • On peut toujours bricoler pour arriver à un résultat, mais le mieux est de savoir ce que l’on fait !
